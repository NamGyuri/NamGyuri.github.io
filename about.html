HOW
I WORK

Pain point

사용자 여정 등에서 사용자가 니즈를 충족할 수 없도록 막는 모든 UX 문제를 살펴 보고 사용자가 겪는 문제점을 확인하여 UI.UX 디자인으로 해결할 수 있는 잠재적 문제를 찾는 단계

Solution

Pain point로 도출한 문제를 기반으로 사용자의 니즈를 충족할 수 있도록 해결책을 제시하고 보완하는 단계

Persona

가상의 사용자를 구성하여 사용자가 겪는 불편을 확인하고, 사용자가 겪는 문제를 간접적으로 체험하면서 해결방안을 계획하는 단계

Scenario

가상의 사용자가 문제를 해결하는 상황을 만들어 문제가 해결 되는 과정을 분석하여 앞서 계획한 해결방안을 검토하고 보완하는 단계

Information Architecture

서비스의 전체 그림과 흐름을 결정하는 콘텐츠 구조도로 사용자에게 언제 어떠한 방식으로 정보를 제공할지를 결정하는 단계

Flowcharts

서비스를 사용하기 위해 수행하는 일련의 단계와 결정을 한눈에 파악할 수 있도록 도와주기 위해 시각적으로 표현하는 단계

Wireframe

사용자 인터페이스 및 핵심 기능을 파악할 수 있도록 선과 도형으로 간단한 형태(프레임)을 만들어 앱의 기능을 시각화하는 단계

Task

서비스를 이용하는 과정을 단계별로 간단히 나열하는 단계

Design

색상, 타이포그래피, 이미지 및 기타 시각적 요소를 선택하여 인터페이스를 만드는 단계

Prototype

구체화되지 않은 서비스의 설계 또는 성능, 전체적인 기능을 간략한 형태로 구현하여 구현 가능성 등을 평가하는 단계

본 사이트는 포트폴리오 용도로 제작된 사이트입니다. 일부 콘텐츠의 저작권은 해당사에 있음을 밝힙니다.
